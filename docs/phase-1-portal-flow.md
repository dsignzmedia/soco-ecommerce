2. Parent Login / Student Registration Flow
-------------------------------------------

### Login / Registration

1. Parent Login: existing guardians authenticate.  
2. New User Registration: capture parent contact + credentials.  
3. Student Profile Creation: prompt immediately after registering.  
4. Add Multiple Students: optional repeating wizard.  

### After Login – Accessible Modules

- School List tied to parent’s region or institution.  
- Uniform Products filtered by school parameters.  
- Product Customization (logos, embroidery) when enabled.  
- Size Guide for each garment.  
- Add to Cart once a student + product combination is valid.  

### Cart & Checkout Sequence

1. View Cart (line items per student).  
2. Edit Cart (quantities, sizes, remove).  
3. Proceed to Checkout (visible only once each student profile is complete).  
4. Enter Address & Contact Details.  
5. Select Payment Method (card, wallet, COD as defined later).  
6. Order Confirmation screen with summary + email.  

### Critical Notes

- A student profile must be finished (School, Grade, Section, Gender) before purchasing.  
- Parents can register multiple students under one account; cart groups items per student.  
- “Proceed to Checkout” appears only when all required student attributes + size selections are set.

4. My Orders – Process Flow
---------------------------

### My Orders Page

- List + search all historical and active orders.  
- Toggle filters: Active Orders / Completed Orders.  

### Order Details View

- Order Number  
- Order Date  
- Product Details (per student)  
- Price Summary (items, tax, delivery)  
- Delivery Address  
- Payment Method  
- Order Status timeline  

### Order Status Stages

1. Order Placed  
2. Order Accepted  
3. Order Packed  
4. Out for Delivery  
5. Delivered  
6. Completed  

### Exchange / Refund Process

1. User clicks **Request Exchange / Refund** from an eligible order line.  
2. Selects Reason from dropdown (size issue, defect, wrong item, other).  
3. Uploads Photograph (optional but recommended).  
4. Submits request and receives on-screen + email confirmation.  
5. Status transitions to **Exchange Requested** or **Refund Requested** and tracks under My Orders.  
6. Company policy governs approval; portal + email provide updates.  
7. Customer care contact information remains accessible for assistance.  

### Policy Notes

- Requests follow the official Return / Exchange guidelines.  
- Approval + logistics updates occur via portal notifications and/or email.  
- Support hours align with store business hours noted on the homepage.
